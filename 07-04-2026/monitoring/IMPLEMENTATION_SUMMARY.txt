Scénario 1: Détection d'Anomalie P&L
────────────────────────────────────

  1. Anomaly detector notice écart > 2σ
  2. Alerte CRITICAL générée: "Sharp decline detected"
  3. Handler déclenché (email/Slack/webhook)
  4. Dashboard mis à jour en temps réel
  5. Recommendation de maintenance générée

Scénario 2: Model Drift
──────────────────────
Performance du modèle se dégrade:
  1. Drift detector calcule 3 scores (data/model/concept)
  2. Score dépassé seuil > 0.6
  3. Alerte générée + retraining scheduled
  4. Maintenance engine crée tâche haute priorité
  5. Health score diminue

Scénario 3: Ressources épuisées
──────────────────────────────
Mémoire > 90%, CPU > 85%:
  1. Quick metrics collection détecte
  2. Alertes générées (CRITICAL)
  3. Maintenance engine schedule cleanup
  4. Recommendations prioriser redémarrage
  5. Dashboard affiche état dégradé

Scénario 4: Backtest Optimization
─────────────────────────────────

  1. Profiler decorators collectent timings
  2. Identifient les fonctions lentes

  4. Recommendations d'optimisation
